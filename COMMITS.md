# Commit Guidelines

## Format

```
<type>: <short description>
```

## Types

- `feat` ‚Äî new feature
- `fix` ‚Äî bug fix
- `chore` ‚Äî build, deps, config, tooling
- `refactor` ‚Äî code change with no behavior change
- `style` ‚Äî formatting, naming
- `docs` ‚Äî documentation only
- `test` ‚Äî tests only

## Rules

- Keep the subject line short (50 chars or less)
- Lowercase, no period at the end
- No AI tool references in commit messages
- No filler words ("update", "change", "misc")

## Examples

```
feat: add fan profile storage
fix: handle missing api key gracefully
chore: add esbuild config
refactor: split prompt builder into helpers
```
